# CLAUDE.md — AI Academy

Project context for Claude Code. Read this first.

## What this is
A **teaching monorepo** (academic setting) showing how Microsoft Agent Framework (MAF) agentic
services compose into **scenario chains** that move business KPIs. Domain: the 9 **HR Service
Delivery (Tier-0)** scenarios (hr-hrsd-01 … 09). Distilled from the production
`C:\code\scenario_chain` platform — the canonical 24-step chain, the step-16 HITL gate, and the
14-field LEDGER concept come from there and must stay consistent with it.

## Dev setup

uv sync                       # or: pip install -e packages/* -e apps/academy-cli

## Architecture
- `packages/academy-core` — the chain runtime. `models.py` (roles, HITL modes, specs),
  `chain.py` (canonical 24-step backbone), `runtime.py` (ChainRunner: executes the six-role
  W2 segment, enforces the step-16 gate, writes the ledger), `ledger.py` (14-field rows),
  `personas.py`, `registry.py`.
- `packages/academy-services` — AI-service adapters, **mock-first**: `hub.py` (ServiceHub),
  `foundry.py` (AOAI chat + polish), `maf.py` (MAF Agent runtime), `search.py` (local keyword
  index over data/policies, Azure AI Search live), `language.py`, `translator.py`,
  `documents.py`, `data.py` (employees/cases/routing).
- `packages/academy-scenarios-hrsd` — one module per scenario; each exposes `SPEC` +
  stage handlers + `SCENARIO`, registered on package import.
- `apps/academy-cli` — the `academy` console command (list/show/run/chain/kpi).
- `apps/academy-api` — FastAPI backend for the M365 Copilot declarative agent. The
  operationIds (listScenarios/getScenario/runScenario) are load-bearing: they must match
  `m365/appPackage/ai-plugin.json` functions and `openapi.json` (enforced by
  tests/test_m365_package.py).
- `m365/appPackage` — declarative agent manifest (v1.7) + API plugin manifest (v2.4) +
  curated OpenAPI (keeps the `__ACADEMY_API_BASE_URL__` placeholder; `m365/package.ps1`
  stamps it). Icons are generated by `scripts/make_icons.py`.
- `infra/` — Bicep + `deploy.ps1` for Azure Container Apps (test on the Deloitte tenant,
  then the customer's subscription — see docs/07).
- `data/` — synthetic HR data only. Never put real employee data here.
- `scenarios/hr-service-delivery/` — markdown teaching briefs, one per scenario.

## Invariants (non-negotiable)
- Every scenario keeps the six-role chain (Assess → Classify → Quantify → Decide → ★HITL(16) →
  Act → Learn) and exactly one gate at canonical step 16.
- The ledger row has exactly 14 fields; every executed step writes one.
- Mock mode must stay deterministic and credential-free — CI and student laptops rely on it.
- All KPI / currency figures are synthetic reference figures, never client claims.
- Adding a scenario = new module in `academy-scenarios-hrsd` + brief in
  `scenarios/hr-service-delivery/` + it must pass the parametrized end-to-end test unchanged.

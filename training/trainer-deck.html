Block 1 · The core idea
15 min

The chain is the unit of value — not the model

Business objective  →  Use case      →  Persona          →  Scenario chain        →  KPI moved
"cut HR ticket load"   policy Q&A        Raj (employee)      Assess→Classify→          ticket deflection
                                          Sofia (HR Ops)      Quantify→Decide→          0% → 55%
                                                              ★HITL(16)→Act→Learn

An AI service that answers a question is a demo. A scenario chain is a production system: agents in named roles, on governed data, gated by a human where it matters, leaving evidence at every step, rolling up to a KPI a named persona owns.

If you cannot fill all five slots of the line above, the scenario is not ready to build. That sentence is the course's thesis — return to it in every block.

Trainer notes — say this "Every failed GenAI pilot you've seen skipped a slot: no named KPI owner, no gate, no evidence. We're going to build the whole line, and by the end of the day you'll ask Copilot an HR question and watch all five slots light up." Common pushback: "isn't this just RAG?" Answer: RAG is one seat (CLASSIFY's grounding) in a seven-stage governed process — §8 shows the other six.
Demo script academy run hr-hrsd-01 on the podium machine, full-screen terminal. Don't explain anything yet — just let them see the trace: steps 11→18, a gate, an answer with a citation, a KPI table. "By 10am you'll be able to read every line of this."

Block 2 · The runtime
35 min incl. Lab 1

Six roles, one gate, fourteen fields

ASSESS(13) → CLASSIFY(14) → QUANTIFY(15) → DECIDE(16) → ★ HITL gate(16) → ACT(17) → LEARN(23)
  • Assess — read the scoped context: this employee, this case. Never the table.
  • Classify — identify what this is: which policy, which intent, which case.
  • Quantify — compute what applies: balances, exceptions, deadlines, confidence.
  • Decide — compose the proposal. Its confidence feeds the gate.
  • ★ Gate — four modes: ZERO_TOUCH ACK_ONLY ESCALATION HITL
  • Act — the only stage allowed to mutate anything, and only after the gate.
  • Learn — gaps, telemetry, retraining signals close the loop.

Every executed step writes one 14-field LEDGER row: run_id, seq, timestamp, scenario_id, step, stage, agent, persona, action, detail, confidence, hitl_mode, actor, outcome. If a step didn't write a row, it didn't happen.

Trainer notes Open packages/academy-core/src/academy_core/runtime.py on screen — ~200 lines. Point at _gate(): ACT is structurally unreachable without an approved gate; the runtime guarantees it, not the scenario author. That's the governance answer clients ask for. Frequent question: "why exactly one gate?" — zero gates = unaccountable; five gates = the humans become the bottleneck the chain was built to remove. The mode, not the count, carries the risk dial.

Block 4 · Architecture walkthrough 1/3
15 min

Layers 1–2: sources → OneLake medallion → serving stores

  • Sources (A1/A2): Workday HCM (C1, SOAP/REST batch), SAP SuccessFactors (C2, OData deltas), ServiceNow HRSD — cases arrive twice: real-time webhooks into Fabric Eventstream (C3) and a nightly reconciliation via the Table API (C4).
  • Bronze (C5/C6): raw + ingest metadata, append-only Delta, 90-day retention. The only zone that ever holds un-tokenized PII.
  • Silver (C7): canonical HRML entities — worker, case, policy doc — PII tokenized at canonical step 4 (Spark calls AI Language, C10), SCD2 history.
  • Gold (C8): the agent contract — gold_hr_worker_v2, gold_hr_case_v1 (scoped views + row-level security), the KB source library, and the Direct Lake KPI model.
  • Serving stores: Azure AI Search (hr-knowledge-base index, HNSW content_vector) and the PostgreSQL LEDGER (append-only, 7y).
Trainer notes The one sentence to land: "nothing at inference time reads Bronze or Silver — the Gold contract is the only agent surface." In the academy repo, data/employees.json and data/cases.json ARE gold_hr_worker_v2/gold_hr_case_v1, and data/kb/ is the KB library — students have been using this layer all morning without knowing it.
Block 4 · Architecture walkthrough 2/3
15 min

Layers 3–4: AI compute → the agent runtime

  • Fabric Spark — the ELT engine (C9 control): Bronze→Silver→Gold merges, the PII tokenize step (C10), and the nightly ledger→Gold KPI rollup.
  • KB chunk & embed — this repo's academy kb pipeline as a Fabric notebook: reads the Gold KB library (C11), embeds via text-embedding-3-small (C12), uploads chunks + vectors to AI Search (C13).
  • Azure OpenAI (Foundry) — two deployments: embeddings and gpt-4.1-mini chat for the DECIDE polish (C15).
  • Azure Container Apps runtime — four sub-components: academy-api (FastAPI; its operationIds ARE the Copilot plugin functions), ChainRunner + MAF agents (Sequential pattern, six roles, gate at 16), the MCP tool server (governed capability registry: gold views C18, search C14, doc-gen), and the HITL gate controller (Adaptive Cards round-trip C21).
  • Writes: every step appends to the LEDGER (C17); approved ACTs write back to ServiceNow (C19) — the platform never becomes a shadow SOR.
Trainer notes — the LLM seat Emphasize where the LLM is not: facts, figures, citations, and deadlines are composed deterministically; the LLM polishes a finished draft (C15) and never invents. Mock mode (polish = identity function) proves the chain works with the LLM removed — run any scenario and show it. Q: "Why MCP and not direct SDK calls?" A: MCP is the governed registry — step 7 of W1; tools are discoverable, permissioned, and auditable, and the same tools serve any agent runtime.
Block 4 · Architecture walkthrough 3/3
15 min

Layers 5–6 + one record end to end

Governance & consumption

  • Entra ID — three identity planes: users (OIDC/OBO, C24/C25 — makes row-level security real), agents (least-privilege app roles), services (managed identities).
  • Purview — scans the OneLake estate (C29), labels ride into KB chunks; runtime pushes lineage + doc-gen audit (C28).
  • Monitor — OTel with run_id correlation (C27): one Kusto query joins traces to ledger rows.
  • Consumption — Copilot (C20), dashboard (C22), Adaptive Cards (C21), Power BI Direct Lake (C23).

Trace: Raj asks about PTO carryover

  1. Workday batch landed him months ago (C1→C5→C7→C8); the PTO policy was chunked, embedded, indexed (C11→C12→C13).
  2. Raj asks Copilot → plugin calls runScenario (C20) under his identity.
  3. ASSESS reads one row (C18) · CLASSIFY retrieves the policy chunk (C14) · DECIDE composes + polishes (C15).
  4. ZERO_TOUCH gate approves; 10 ledger rows written (C17); lineage (C28), traces (C27).
  5. Answer lands in Teams. No ticket exists — the rollup moves the deflection KPI in Power BI (C23).

Block 5 · Grounding
45 min incl. Lab 4

The knowledge base: corpus → chunks → vectors → hybrid retrieval

  • Corpus: 30 synthetic documents in data/kb/ — 15 policies, 10 guides, 3 FAQs, 2 localized (es/de). Frontmatter metadata on every doc: doc_type, category, jurisdictions, language, scenarios — the last one records which chains a document grounds; a test fails if any scenario loses coverage.
  • Chunker: splits at ## sections → 132 chunks, each a search record with a content_vector column.
  • Embeddings, two modes, one interface: deterministic 256-dim feature-hash offline (real cosine vector search, zero credentials) ↔ text-embedding-3-small (1536-dim) live.
  • Retrieval, three strategies: local keyword (deterministic, drives all tests) · local vector (--vector) · Azure AI Search hybrid (BM25 + kNN/HNSW), graceful fallback. academy kb push provisions the index.
  • Two deliberate gaps: garnishment topics are kept OUT of the KB (so the hr-hrsd-08 escalation demo keeps firing — regression-tested), and sensitive topics are route-to-a-human entries only.
Trainer notes — the honesty beat The mock embedding is soft-lexical, not semantic. When a student's --vector result looks odd, that IS the lesson: hash embeddings rank by shared vocabulary; true semantic wins ("mother is ill" → caregiver policy with zero shared words) need the live deployment. Don't oversell the mock — its job is teaching the pipeline mechanics offline.
Block 4½ · Orchestration patterns
15 min

MAF orchestration: step 12 is a design choice

PatternShapeUse whenHR area
SequentialA → B → C, one batonKnown pipeline; each step needs the lastHR Service Delivery (this module), Onboarding
ConcurrentA ∥ B ∥ C, fan-inIndependent sub-tasks, merged resultsLearning & Skills
Group Chatdebate + arbiterAnswer benefits from challengePerformance & Goals
HandoffA transfers to BDistinct ownership phasesTalent Acquisition, ER
Magenticmanager decomposesOpen-ended, unscripted asksWorkforce Planning

Rule of thumb: pick the least coordination the use case survives. Tier-0 requests are short, high-volume, well-understood — Sequential wins, and the sophistication budget goes to grounding and the gate.

Block 8 · Capstone
20 min brief

Capstone: ship scenario hr-hrsd-10

Students pick an uncovered Tier-0 request (holiday lookup, expense Q&A, org-chart "who do I ask", onboarding buddy FAQ…) and ship it end to end:

  1. ScenarioSpec — personas, synthetic KPIs, Azure service seats, HITL mode defended in the module docstring.
  2. Six stage handlers + any new synthetic data under data/ (KB doc with correct frontmatter, including scenarios: coverage).
  3. A brief in scenarios/hr-service-delivery/ matching the house format.
  4. Registration in academy_hrsd/__init__.py — the parametrized contract tests must pass unchanged.

Grading rubric

DimensionWeightWhat "excellent" looks like
Chain correctness40%One gate at 16 · 14-field rows · deterministic mock · honest confidence
Business case30%The five-slot line complete: objective→use case→persona→chain→KPI
Code quality & tests20%ruff clean; handlers read like the existing nine
The brief10%Walkthrough, KPIs, Azure seats, labs — a teachable artifact
Trainer notes The acceptance bar is mechanical on purpose: tests/test_chain_runtime.py is parametrized over the registry — a new scenario is automatically held to every invariant. The most common failure: gate mode chosen by vibes. Push them to name the concrete harm scenario that justifies anything above ZERO_TOUCH.
